# Contributing to WeClaw

Thank you for your interest in contributing. WeClaw is open source and community-driven.

## How to Contribute

### Code
- Fork the repo, create a branch, submit a PR
- Follow existing code style (see project conventions)
- Add tests for new behavior when applicable
- Reference any related Issues in your PR

### Documentation
- Fix typos, clarify unclear sections
- Add examples or use cases
- Improve README, ARCHITECTURE.md, or platform guides

### Skills
- Create or tune markdown skill files in `skills/`
- Share playbooks for different group types (business, family, community)

### Bug Reports & Feature Requests
- Use [GitHub Issues](URL)
- Tag with `bug` or `enhancement`
- Include steps to reproduce, environment, and expected vs. actual behavior

### Good First Issues
- Look for `good-first-issue` label
- Ideal for new contributors

## Code of Conduct

We follow a code of conduct that ensures a respectful, inclusive environment. See [CODE_OF_CONDUCT.md](URL).

## Questions?

Open a [Discussion](URL) or comment on an Issue. We're happy to help.
